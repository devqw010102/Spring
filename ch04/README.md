### 데이터 통신 방법

<pre>
  1. 동기식
  2. 비동기식
</pre>

#### 동기식
<pre>
  먼저 시작된 하나의 작업이 끝날 때까지 다른 작업을 시작하지 않고 기다렸다가 다 끝나면 새로운 작업을 시작하는 방식

  ex) 로그인 화면 -> 로그인 프로세스 처리    == 로그인 화면 반드시 필요
</pre>

#### 비동기식(Restful API)
<pre>
  먼저 시작된 작업이 있더라도 완료 여부와 상관없이 새로운 작업을 시작하는 방식

  ex) URL 주소 -> 프로세스 처리
</pre>

#### HTTP Method
| Method | Fn |
| ------ | ---- |
| GET | 조회 |
| POST | 생성 |
| PUT | 수정 |
| DELETE | 삭제 |

#### HTTP 상태 코드
| HTTP 상태 코드 | 의미 |
| ------------ | ------ |
| 200 OK | 요청을 정상적으로 처리 |
| 201 Created | 요청에 따라 자원을 정삭적으로 생성 |
| 400 Bad Request | 요청 정보가 잘못되었을 때 |
| 401 Unauthorized | 요청을 처리하기 위해 인증이 필요할 때 |
| 403 Forbidden | 요청에 대한 권한이 없을 때 |
| 404 Not Found | 요청의 대상 자원이 없을 때 |
| 500 Internal Server Error | 서버 오류가 발생했을 때 |
